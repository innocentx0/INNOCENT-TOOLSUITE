## 🛠️ Tool Overview

### `orchestrator.sh` — The Conductor
Designed to run as a **daily cron job** (default: 3:00 AM). It sequentially invokes `getsub.sh` and `ghostsub.sh`, ensuring the recon pipeline kicks off automatically every day without manual intervention.

```bash
# Add to crontab:
0 3 * * * /path/to/Conductor/orchestrator.sh
```

---

### `getsub.sh` — Subdomain Enumeration
Takes a file containing a list of domains and runs parallel enumeration (up to 50 threads) using `subfinder`. Results are validated through `httpx` (filtering out 404/503) and saved to `../Ghostsub/sublist.txt` for the next stage. It also runs `gospider` to scrape additional subdomains from JS files and internal links.

```bash
./getsub.sh domains.txt
```

**Input:** `.txt` file with one domain per line
**Output:** `../Ghostsub/sublist.txt`

**Dependencies:** `subfinder`, `httpx`, `gospider`, `anew`

---

### `ghostsub.sh` — Subdomain Takeover Detection
Reads `sublist.txt` and for each subdomain:
1. Sends an HTTP request and matches the response against a dictionary of known takeover fingerprints (unclaimed S3 buckets, missing GitHub Pages, unconfigured Heroku/Shopify/Vercel, etc.)
2. Resolves **CNAME** records using `dig` and checks whether the CNAME target is also vulnerable to takeover
3. Diffs today's results against yesterday's — only **new** findings trigger a Discord notification

```bash
./ghostsub.sh
```

**Input:** `./sublist.txt`
**Output:** `./loots/YYYY-MM-DD.vuln.results`, `./loots/YYYY-MM-DD.cname`, `./loots/YYYY-MM-DD.cname.results`
**Notifications:** Discord via `notify`

**Dependencies:** `curl`, `dig`, `anew`, `notify`, `gobelly`

---

### `automatic.sh` — Full Single-Domain Pipeline
All-in-one recon tool for deep reconnaissance on a specific domain. Runs the following steps in order:

| Step | What it does |
|------|-------------|
| Subdomain enumeration | `subfinder` + `httpx` |
| IP resolution | `host` for each subdomain |
| Port scan | `masscan` on ports 80/443 |
| Nuclei scan | CVE, exposure, panel templates |
| Wayback | `waybackurls` for historical assets |
| 401/403 bypass | Python script `authslicer.py` |

```bash
./automatic.sh target.com
```

**Dependencies:** `subfinder`, `httpx`, `nuclei`, `anew`, `masscan`, `waybackurls`, `paramspider`, `notify`, `gobelly`

---

### `bypasser.sh` — 401/403 Bypass
Takes a list of URLs returning 401 or 403 and attempts bypass via headers like `X-Forwarded-For: 127.0.0.1`. If the response flips to 200, the domain is saved to the output file.

```bash
./bypasser.sh forbidden_list.txt bypass_results.txt
```

**Dependencies:** `curl`, `anew`, `gobelly`

## ⚙️ Initial Setup

### 1. Clone the repo and set up folders

```bash
git clone https://github.com/yourusername/conductor.git
cd conductor
mkdir -p Getsub Ghostsub Bypasser
```

### 2. Create the `.env` file

In the **project root** (one level above `Ghostsub/`), create the `.env` file:

```bash
# .env

# Discord webhook URL (e.g. https://discord.com/api/webhooks/...)
DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/XXXXXXXXXX/XXXXXXXXXX

# Username shown in Discord messages
DISCORD_USERNAME=Conductor-Bot

# Discord channel name or ID (used as a label in the config)
DISCORD_CHANNEL=bug-bounty-alerts
```

> ⚠️ **Never commit `.env` to Git.** Add it to `.gitignore` right away:
> ```bash
> echo ".env" >> .gitignore
> ```

### 3. About `provider-config.yaml`

Already set up to read variables from `.env`. It is consumed by `notify` (ProjectDiscovery) to dispatch Discord messages. No manual edits needed as long as `.env` is correct.

### 4. Install dependencies

```bash
# Go tools (ProjectDiscovery)
go install github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest
go install github.com/projectdiscovery/httpx/cmd/httpx@latest
go install github.com/projectdiscovery/nuclei/v3/cmd/nuclei@latest
go install github.com/projectdiscovery/notify/cmd/notify@latest
go install github.com/jaeles-project/gospider@latest
go install github.com/tomnomnom/anew@latest
go install github.com/tomnomnom/waybackurls@latest

# masscan (requires sudo)
sudo apt install masscan -y
```

### 5. Make scripts executable

```bash
chmod +x orchestrator.sh automatic.sh
chmod +x Getsub/getsub.sh
chmod +x Ghostsub/ghostsub.sh
chmod +x Bypasser/bypasser.sh
```

## 🚀 Quick Usage

### Automatic mode (daily cron)
```bash
# Run manually to test
./orchestrator.sh

# Or schedule via crontab at 3:00 AM
crontab -e
# → 0 3 * * * /absolute/path/to/conductor/orchestrator.sh
```

### Full recon on a single domain
```bash
./automatic.sh example.com
```

### Subdomain enumeration only
```bash
echo "example.com" > targets.txt
./Getsub/getsub.sh targets.txt
```

### Takeover detection only
```bash
# Make sure Ghostsub/sublist.txt exists first
./Ghostsub/ghostsub.sh
```

### 401/403 bypass only
```bash
./Bypasser/bypasser.sh forbidden_urls.txt results.txt
```

## 📋 Useful Input Files

The `wildcard.txt` file contains wildcard domains from public bug bounty programs (source: [bounty-targets-data](https://github.com/arkadiyt/bounty-targets-data/blob/main/data/wildcards.txt)) and can be fed directly into `getsub.sh` to kick off large-scale recon on already in-scope targets.

```bash
./Getsub/getsub.sh wildcard.txt
```
